Tables in HTML

Making tables straight into HTML via a text editor in an html file is pretty easy but also rather laborious. By the HTML default, they lack lots of features and styling that you might expect. Styling them doesn't take too long, but if you need features like filtering, reordering or grouping; then it's probably best to use a component from a library or framework such as jQuery or React.

Being Efficient

I recommend heavy use of copy and paste to save typing. When adding an entire new table, copy the template below. When adding a row, just copy the last row of whatever table you're adding to and change the details as needed. To add multiple rows copy the last row, remove details, copy the new row, paste until you have all your rows, then fill in the details. Adding an extra collumn to a table, particularly in the middle, is kind of a pain because you have to go though every collumn adding a cell in the correct place and make sure that they match up.

Typically people want table headers making up all the cells in the first row, but HTML does not limit you on that. It's usually a good idea to get your headers decided, before putting in the rest of the info, as this helps you know what order the collumns will be


Table Tags explained

4 main tags make up a table.

  • "table" which literally defines a table, and will have table rows inside.
  • "tr" which is short for "table row", they go inside the table tag. You have one tag per row of the table including header row
  • "th" which is short for "table header" and are a cell of the table that should be treated as a header - the HTML default is that text in them is bold and centered.
  • "td" which is short for "table data". These are the main cells in a table. They go inside "tr" tags, in the order you want them displayed - the top td tag within a tr is the first on the left of that row. Note that to avoid having wonky tables, you need to make sure that there are the same total number of tr's and th's in each collumn.
